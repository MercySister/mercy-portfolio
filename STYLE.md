# Mercy Portfolio 样式指南

## 1. 文档定位

本文档总结当前网站已经形成的视觉语言与实现规则，用于后续新增页面、组件或内容时保持一致。

当前设计可概括为：

> 深色系统界面 × 米白实体票券 × 电蓝状态信号

它不是常规的卡片式作品集，也不是纯粹的赛博朋克界面。核心是把个人履历呈现为一组可以被读取、核验和归档的“票券 / 档案 / 系统记录”。

## 2. 设计关键词

- Ticket：票券、存根、打孔边、条码和序列号。
- Archive：档案编号、状态、标签、分区记录。
- System：在线状态、访问授权、系统监测和数据读数。
- Editorial：强标题、克制留白、衬线体点缀。
- Technical：等宽字体、全大写、字距和编号体系。
- Restrained：以黑、米白、灰为主，电蓝只承担强调作用。

## 3. 设计原则

### 3.1 强对比，少颜色

深色页面背景承载环境感，主要内容放在米白票券上。电蓝只用于当前状态、选中、关键标签和交互反馈。

### 3.2 直角优先

全局圆角基准为 `0`。卡片、按钮、输入框和标签均以直角为主。圆形只用于状态点、蓝色标记和极小的视觉信号。

### 3.3 信息像档案，不像营销页

优先使用编号、标签、时间、状态和模块化正文。避免常规营销网站的渐变大标题、圆角玻璃卡片和装饰性 CTA。

### 3.4 装饰必须服务主题

虚线、点阵、打孔边、条码和状态灯都应强化票券与系统档案主题。不要引入与该叙事无关的插画或纹理。

### 3.5 动效克制但持续有生命感

背景波形和状态灯提供持续动态；内容切换和悬停反馈保持短促。不要让多个高强度动画同时争夺注意力。

## 4. 色彩系统

### 4.1 核心色板

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| 页面背景 | `#0e0e0e` | 全站深色背景 |
| 深色正文 / 墨色 | `#0a0a0a` | 票券内标题、选中按钮、主要文字 |
| 主票券 | `#f2f0ed` | 一级内容卡片和弹窗 |
| 内层票券 | `#f7f5f2` | 卡片中的职责、能力和信息分组 |
| 页面前景白 | `#f0f0f0` | 深色背景上的主要文字 |
| 电蓝 | `#1500ff` | 当前状态、强调点、进度和关键标签 |
| 互动粉色 | Tailwind `pink-400/500` | 已点赞状态，仅用于点赞 |

### 4.2 中性色透明度

票券上的文字主要使用黑色透明度建立层级：

- 主要标题：`black/80` 至 `black/82`；
- 正文：`black/58` 至 `black/62`；
- 次要信息：`black/32` 至 `black/52`；
- 编号和装饰：`black/10` 至 `black/28`；
- 分隔线和边框：黑色 `8%` 至 `20%`。

深色背景上的内容主要使用白色透明度：

- 底栏主要操作：`white/65`，悬停提升至 `white/90`；
- 次要状态：`white/30` 至 `white/35`；
- 边框与轨道：白色 `6%` 至 `12%`；
- 两侧竖排文字：白色 `18%`。

### 4.3 电蓝使用规则

电蓝应限定在以下场景：

- 当前或在线状态；
- Hero 蓝色圆点；
- 顶部滚动进度；
- 蓝色标签和状态描边；
- 可交互元素的悬停反馈；
- 访问量图标与数字；
- 滚动条悬停。

不要把电蓝用于大面积背景，也不要增加第二个竞争性的品牌主色。

## 5. 字体系统

### 5.1 字体家族

| 角色 | 字体 | 用途 |
| --- | --- | --- |
| 中文与正文 | `Noto Sans SC` | 中文正文、默认界面字体 |
| 英文标题 | `Space Grotesk` | Hero、公司名、项目名、重要标题 |
| 技术与元数据 | `Space Mono` | 标签、编号、状态、时间、按钮、计数 |
| 编辑性点缀 | `Playfair Display` | Hero 中的斜体 “Access” |

默认字体栈：

```css
--app-font-sans: 'Noto Sans SC', 'Space Grotesk', sans-serif;
--app-font-mono: 'Space Mono', monospace;
--app-font-serif: 'Playfair Display', serif;
```

`index.html` 目前还加载了 `Inter`，但现有主要样式未使用它。新增界面不应将 Inter 作为新的主字体；后续可删除该冗余引用。

### 5.2 字体层级

| 层级 | 当前表现 |
| --- | --- |
| Hero 姓名 | `Space Grotesk`，Black，`clamp(42px, 8vw, 72px)`，紧字距 |
| 项目 / 教育主标题 | `Space Grotesk`，Black，约 30px |
| 公司名 / 模块标题 | `Space Grotesk`，Bold，约 18–24px |
| 正文 | 14px，常规字重，约 1.6 行高 |
| Section Label | `Space Mono`，约 10.9px，字距 `0.18em`，全大写 |
| 标签 / 状态 | `Space Mono`，约 10.9px，字距 `0.08–0.14em` |
| 微型元数据 | `Space Mono`，8–10px，宽字距，全大写 |

### 5.3 排版规则

- 英文状态、标签和系统文案使用全大写。
- 正文保持正常大小写，不使用大段全大写。
- 标题优先短句，避免过长导致票券结构失衡。
- 编号统一使用两位格式，如 `01`、`02`。
- 系统语义之间可使用 `//` 分隔。
- 中文正文保持清晰和自然，不强制使用等宽字体。

## 6. 页面结构

### 6.1 全局框架

- 页面背景为固定深黑底，并叠加极低透明度的水平波纹纹理。
- 全屏 Canvas 绘制缓慢的白色丝带波形，位于内容下方。
- 左右各有一条 28px 宽的竖向状态栏，文字旋转 90 度。
- 顶部固定 2px 滚动进度条，进度使用电蓝。
- 底部固定操作栏，深黑半透明背景并带 16px 模糊。

### 6.2 内容容器

- Hero 与正文内容最大宽度均为 Tailwind `max-w-4xl`，约 896px。
- 内容整体居中。
- 页面当前左右预留约 28px / 36px，底部预留约 72px，避免被底栏遮挡。
- 主要内容区域以 16px 左右的模块间距组织。

### 6.3 栅格

- 默认移动优先，窄屏使用单列。
- `md`（768px）及以上，多数信息卡切换为两列。
- 兴趣卡片移动端两列，`md` 及以上五列。
- Hero 和卡片标题区域允许换行，元数据可右对齐。

## 7. 核心组件

### 7.1 Ticket Card

一级内容容器使用 `.ticket-card`：

- 背景：`#f2f0ed`；
- 文字：`#0a0a0a`；
- 无圆角；
- 默认无阴影；
- `overflow: hidden`；
- 常用内边距为 24–28px。

卡片不应默认做成浮层。只有经历卡等明确可交互或需要强调的内容才使用轻微悬浮阴影。

### 7.2 Inner Ticket Card

二级信息容器使用 `.ticket-card-inner`：

- 背景：`#f7f5f2`；
- 1px、黑色 8% 边框；
- 用于能力、职责、结果和教育信息分组；
- 不使用额外圆角或重阴影。

### 7.3 Perforated Edge

`.perf-right` 在卡片右侧保留 28px 票根区域，并以 1px 黑色 18% 虚线分隔。

适用于 Hero 和重点项目，不应应用于每一张卡片。

### 7.4 Dotted Separator

`.dot-sep` 使用连续中点形成分隔线：

- 字体使用 `Space Mono`；
- 字号约 11px；
- 黑色 20%；
- 用于票券内部结构分区。

它比普通实线更能表达纸质票券，应优先用于一级区块分隔。

### 7.5 Section Label

`.section-label` 用于模块标题上方的档案标签：

- `Space Mono`；
- 全大写；
- 约 11px；
- 字距 `0.18em`；
- 黑色 42%；
- 下方间距 8px。

文案可采用 `01 // Identity_Record` 形式。

### 7.6 Tab Button

`.tab-btn` 为直角描边按钮：

- 默认透明背景、黑色 16% 边框；
- `Space Mono`、全大写、宽字距；
- 内边距约 `7px 18px`；
- 选中态为黑底米白字；
- 未选中悬停仅增加浅灰背景。

不要将页签改成圆角胶囊。

### 7.7 Tag Chip

`.tag-chip` 用于能力、业务领域和项目标签：

- 直角、1px 描边；
- 浅黑背景与中灰文字；
- 小字号等宽字体；
- 横向紧凑排列并允许换行。

`.tag-chip-blue` 仅用于重点标签，使用电蓝 6% 背景、28% 边框和 80% 文字。

### 7.8 Status Pill

`.status-pill` 用于 `ACTIVE`、`CONFIRMED`、`ARCHIVED` 等状态：

- 使用 `currentColor` 控制文字、边框和状态点；
- 左侧 4px 圆形状态点；
- 状态点以 2 秒周期轻微闪烁；
- 不使用填充背景。

### 7.9 条码与序列号

条码由不同宽度与高度的黑色竖条组成，右侧配合等宽序列号。它是票券身份的一部分，适用于重点项目和留言回执，不应作为无意义的页面填充。

### 7.10 表单与弹窗

- 弹窗使用 `ticket-card`，最大宽度 `max-w-md`，内边距约 32px。
- 遮罩为黑色 75%，带轻微背景模糊。
- 输入框为半透明白底、1px 浅黑边框、直角。
- 主提交按钮为黑底白字，悬停降低黑色强度。
- 成功态使用简洁图标、等宽状态文案和条码收尾。

## 8. 图标与图形

- 功能图标统一使用 Lucide，当前包括留言、关闭、发送、访问和点赞。
- 图标尺寸以 12–16px 为主，线性风格，不使用彩色填充图标。
- 点赞成功可以使用粉色填充心形，作为唯一例外。
- 大型视觉图形由 Canvas 波形承担，不额外加入插画风图标。
- 电蓝圆点有 14px 和 22px 两种尺寸，用于视觉锚点而非普通列表项目符号。

## 9. 动效系统

### 9.1 持续动效

- 全屏背景丝带：低透明度、缓慢水平波动。
- Hero 波形：灰阶渐变底上的多层白色曲线与扫光。
- 状态点：2 秒呼吸闪烁。
- 访问量和点赞量：首次读取时约 1.4 秒缓出计数。

### 9.2 进入与切换

| 场景 | 当前参数 |
| --- | --- |
| Hero 进入 | 透明度 + `y: -20`，0.5s，ease-out |
| 页签内容切换 | 透明度 + 轻微上下位移，约 0.25s |
| 卡片进入 | 透明度 + 12px 位移，约 0.3–0.4s |
| 经历列表 | 从左 12px 进入，项目间隔约 0.1s |
| 留言弹窗 | 透明度、24px 位移、0.97 缩放，约 0.24s |
| 兴趣卡片 | 0.92 缩放进入，项目间隔约 0.06s |

### 9.3 悬停与按压

- 普通悬浮卡片：上移 2px，并增加柔和黑色阴影。
- 兴趣卡片：悬停缩放至 1.06，按压缩放至 0.95。
- 标签和按钮只做颜色与背景变化，不使用弹跳。

### 9.4 动效约束

- 新增过渡通常控制在 180–500ms。
- 内容动效优先使用 `ease-out`。
- 同一区域不要同时使用大幅位移、旋转和缩放。
- 持续动画必须保持低对比度，不能影响阅读。
- 当前尚未实现 `prefers-reduced-motion`；后续新增动画时应优先补齐该能力。

## 10. 响应式规则

- 以移动端单列为默认结构。
- `md` 是主要布局切换点，不随意增加多个相近断点。
- Hero 姓名使用 `clamp` 自适应，不设置固定桌面字号。
- 卡片内部标题和元数据使用 `flex-wrap`，避免窄屏挤压。
- 标签容器必须允许换行。
- 固定底栏应保持主要操作、语言切换和统计可见。
- 页面底部必须预留空间，避免内容被固定底栏覆盖。
- 新组件至少检查 320px、768px、1024px 和 1440px 宽度。

## 11. 内容风格

- 中文与英文内容保持同一信息层级，不改变布局结构。
- 标题直接、具体，避免营销化口号堆叠。
- 经历内容按“业务模块 → 职责 / 行动 → 结果”组织。
- 状态词保持简短，如 `ACTIVE`、`ARCHIVED`、`CONFIRMED`。
- 系统微文案可使用下划线和双斜杠，如 `System_Status // Online`。
- 装饰编号和序列号应稳定，不在同一组件中随机变化。

## 12. Do / Don't

### Do

- 使用深黑背景、米白票券和单一电蓝强调。
- 使用直角、细描边、点阵和票根结构。
- 用等宽小字表达编号、状态、日期和标签。
- 保持标题强、正文克制、元数据弱化的层级。
- 新内容优先复用 `.ticket-card`、`.ticket-card-inner`、`.section-label`、`.tag-chip` 和 `.status-pill`。
- 在移动端优先保证阅读顺序，再考虑视觉对称。

### Don't

- 不增加大面积蓝紫渐变。
- 不把卡片改成大圆角或玻璃拟态。
- 不使用厚重、多层、彩色阴影。
- 不在每个元素上添加动效。
- 不混入新的主字体或不相关的图标风格。
- 不把小号等宽文字用于长篇正文。
- 不用纯装饰元素挤压真实内容空间。

## 13. 当前实现注意事项

- 视觉 token 同时存在 CSS 变量、Tailwind 类和行内样式，尚未完全统一。
- 多处使用 `#1500ff` 和黑色透明度行内值；新增组件应优先复用现有 token，避免继续增加近似色。
- `home.tsx` 中部分标题通过行内 `fontFamily` 强制使用 `Space Grotesk`，这是当前既有模式。
- Google Fonts 为外部依赖；加载失败时会回退到系统字体。
- 全屏背景和 Hero 各自使用一个持续运行的 Canvas 动画。
- 当前无圆角并非遗漏，而是票券视觉的明确组成部分。
